# 1. 产品定位

这是一个**旅游攻略制作与管理网站**，核心目标是：

1. 以结构化方式管理一整条旅行攻略。
2. 支持“总攻略 → 子行程/景点/餐饮/预算/交通/待办事项/参考资料/游记/评估与回顾”等多层级内容。
3. 支持富文本、图片、表格、列表、关联引用等内容。
4. 每个页面都可导出 PDF。
5. 最终能够用于：
    - 自己做旅行规划
    - 输出成可打印的攻略文档

从你给的参考图看，整体风格建议是：

- **Notion 风格的信息密集页面**
- 左侧目录导航
- 主体区域卡片化、模块化
- 图文混排
- 表格型预算/行程清单/餐饮/待办事项
- checklist形式的物品清单
---

# 2. 系统目标

系统要解决的不是“写一篇游记”，而是：

- 把一个旅行项目拆成可管理的结构
- 每个模块都可以单独查看、编辑、导出
- 各模块之间通过“关联”形成完整攻略体系
- 后续可以扩展 AI 辅助生成、行程推荐、自动排版 PDF
---

# 4. 功能需求总览

## 4.1 通用能力

所有页面都应具备以下能力：

1. **查看**
2. **编辑**
3. **新增**
4. **删除**
5. **排序**
6. **关联**
7. **上传图片**
8. **富文本编辑**
9. **导出 PDF**

其中：

- 富文本内容支持标题、加粗、列表、引用、代码块、链接、图片嵌入。
- 图片支持上传、替换预览。
- 表格支持增删行、编辑单元格、排序。
- 页面之间支持跳转与引用。

# 5. 页面级详细需求

## 5.1 旅游攻略清单页

### 页面目标

展示所有旅游攻略项目，用户从这里进入某个具体攻略。

### 展示方式

参考你的图，建议采用：

- 图片封面
- 攻略名称
- 计划出游日期
- 状态标签（未出游 / 已出游）

### 功能要求

1. 以卡片形式展示攻略。
2. 每张卡片至少包含：
    - 封面图
    - 攻略名称
    - 计划出游日期区间
    - 状态
3. 支持点击卡片进入单个旅游攻略入口页。
4. 支持搜索：
    - 攻略名称
5. 支持排序：
    - 最近编辑
    - 出行日期
    - 创建时间
6. 支持筛选：
    - 状态
7. 支持新建攻略
    1. 以弹框形式新建

# 6. PDF 导出需求

你要求“每个页面均需要有 pdf 导出功能”，这里建议明确为：

## 6.1 导出范围

以下页面都应支持 PDF 导出：

- 单个旅游攻略入口页
- 单个子行程页
- 餐饮页
- 景点页
- 单个景点页
- 每日交通页
- 物品清单页
- 待办事项页

## 6.2 导出规则

1. 导出时尽量pdf中排版更合理。
2. pdf中用长图，不要考虑分页打印，主要使用场景是在手机端查看。
3. 支持图片压缩和缩放。
4. 表格分页时要避免内容截断。
5. 导出内容必须和当前页面数据一致。

## 6.3 导出格式建议

- 横向表格页面可支持横版
- 封面页单独样式

# 9. 页面跳转与引用规则

系统必须支持：

1. 从总攻略页跳转到某个子行程页。
2. 从子行程页跳转到景点详情页。
3. 从子行程页、景点页、餐饮页跳转到参考资料页对应子行程位置。
4. 所有页面都可返回总攻略页。
---

# 10. 前端交互要求

建议前端具备以下能力：

1. 左侧导航固定。
2. 页面主区域可滚动。
3. 卡片可点击。
4. 表格支持内联编辑。
5. 图片可预览。
6. 支持拖拽排序。
7. 支持加载状态和错误提示。
8. 支持响应式布局。

# 11. 技术建议

## 后端

- Python + FastAPI

## 数据库

- MySQL

## 文件存储

- Docker Compose
- 外部独立 MySQL 数据库
- 后端服务容器化
- 前端由后端托管

# 12. 非功能需求

## 12.1 性能

- 页面首屏尽量快
- 列表支持分页或懒加载
- 图片支持压缩
- PDF 导出支持异步任务

## 12.2 安全

- 登录认证
- 用户分为编辑者和查看者
- 不用专门创建用户和权限管理界面，只需要在数据库中直接管理就行
- 编辑者有页面全部权限
- 查看者只能查看，不能编辑
- 数据隔离
- 文件上传类型限制
- 防止恶意链接

## 12.3 可维护性

- 组件化开发
- 数据模型稳定
- 页面模块可复用
- 导出逻辑独立封装

# 14. 最终验收标准

一个页面如果满足以下条件，可认为需求完成：

1. 页面能正确展示对应内容。
2. 页面能新增、编辑、删除内容。
3. 页面能与其他页面建立跳转/引用。
4. 页面能导出 PDF。
5. 图片、表格、富文本都能正常保存和恢复。
6. 刷新页面后数据不丢失。
7. 移动端或窄屏下至少可阅读。
